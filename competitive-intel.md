## AEM vs Sitecore

**Bottom line: Sitecore moved to Visionaries in 2025. Lead with ecosystem and AI.**

- Sitecore has aggressively pivoted to XM Cloud (composable, headless-first) — mid-transition risk
- **AI**: AEM has Firefly + Sensei + 10 named agents. Sitecore AI is an add-on, behavioral/recommendation-focused, not generative
- **Authoring**: AEM Universal Editor + DA beat Sitecore on author simplicity
- **Personalization gap**: Sitecore personalizes out-of-the-box via xDB. AEM requires Adobe Target. Real weakness if prospect wants CMS + personalization bundled cheaply — don't ignore this
- **Dev talent**: Sitecore .NET devs easier to find and cheaper ($57–$80/hr vs AEM Java specialists)
- **TCO**: AEM 3-year TCO $2.1M–$4M; Sitecore comparable at enterprise scale

**Winning move**: Complete Adobe ecosystem (AEP, Analytics, Target, Firefly all native vs. bolt-on) + EDS performance + Firefly indemnification. Sitecore's XM Cloud migration is still incomplete — exploit the transition complexity risk.

## AEM vs Contentful

**Bottom line: Total stack cost and governance depth. Contentful+Cloudinary = two vendors, two bills, custom bridge.**

- Contentful: ~$500/month entry. AEM: ~$50K/year. Framing matters — per-page, per-author, per-brand calculation often flips this
- Contentful has **no native DAM** — Cloudinary fills it. AEM Assets is fully integrated with Creative Cloud
- AEM Assets: 12.2% DAM mindshare (Gartner 2025). Bynder: 4.8%. Cloudinary: Visionary (not Leader)
- Contentful + Cloudinary = two procurement relationships, two integration points, two support contracts
- AEM Sites + AEM Assets share a single JCR repository, same login, same workflows — no bridge required
- Developer experience: Contentful genuinely wins for API-first, greenfield, developer-led teams (be honest about this)
- Enterprise wins: AEM wins when marketing-led, multilingual, complex governance, CC usage

**Winning move**: Map the prospect's full tool count (CMS + DAM + project management + personalization + analytics). Show what AEM consolidates. Ask about the integration and maintenance cost of point solutions.

## AEM EDS vs Next.js/Vercel

**Bottom line: Same developer experience + authoring + governance + AI. Vercel is an additional cost. EDS is included.**

- EDS: vanilla HTML, CSS, ES6+. No build tooling, no bundler. Push to GitHub → live immediately
- Next.js: two major breaking releases in 12 months (Oct 2023 – Oct 2024). EDS web-standard base is stable by nature
- Next.js requires Node.js server for SSR. EDS is pure CDN — no server to manage
- EDS achieves Lighthouse 100 by default. Next.js requires significant optimization work; React's client-side JS is the key limiter
- EDS load time: ~1 sec. Traditional CMS (including Next.js with heavy components): 3–4 sec
- Developer profile: EDS is frontend-developer territory. No Java/HTL expertise required — massively larger talent pool
- Vercel Enterprise = additional line item on top of custom dev. EDS included in AEM Sites license

**Winning move**: EDS gives the same git-first, JS-first developer experience as Next.js — but with the authoring workflow, governance, DAM integration, analytics, and AI that a custom Next.js app would require you to build from scratch. Reference: [aem.live/blog/edge-next](https://www.aem.live/blog/edge-next).

## Win/Loss Patterns

### AEM Wins When...
1. Already in Adobe ecosystem (Analytics, Target, Creative Cloud, Workfront) — switching cost prohibitive, native integrations already working
2. Complex multi-brand, multilingual, multi-channel at scale — AEM's MSM + permission model + governance has no peer
3. Performance-critical sites — EDS Lighthouse 100 directly maps to SEO rankings and conversion rates
4. Brand/IP-sensitive AI content — Firefly indemnification resolves legal team blockers
5. Content velocity at enterprise scale — GenStudio end-to-end; competitors can't match without bespoke integration
6. AI search (LLMO) early movers — only GA enterprise solution; first-mover advantage window open now

### AEM Loses When...
1. **Price sensitivity** — Drupal ($0), WordPress VIP ($25K), Contentful ($500/mo) win on budget alone; counter with full-stack TCO
2. **Developer-led, headless-only** — Contentful's developer experience genuinely wins when there's no marketer in the buying committee
3. **Personalization-only on a CMS budget** — Sitecore xDB out-of-the-box without needing Adobe Target as a separate SKU
4. **Speed-to-live is #1 criterion** — Optimizely's months vs. AEM's 9–18 month implementation is a real objection; counter with EDS dramatically reducing go-live time
5. **Gartner-driven deals with Optimizely on shortlist** — have the counter-narrative ready before the first call
6. **Mid-market companies** — AEM's $750K–$1M first-year TCO is out of range under ~$500M revenue without a clear ROI model

### How to Handle the Gartner #2 Objection

> "Adobe has been a Gartner DXP Leader for 8 consecutive years — that's the longest streak in the category. Optimizely's top ranking this year reflects their modular, point-solution strategy. When you start pricing out full Optimizely capabilities — CMS, experimentation, personalization, CDN, DAM — the comparison to AEM's bundled pricing often inverts. More importantly: Optimizely has no AI agent suite comparable to Adobe's 10 agents announced this year, no content supply chain connecting brief to publish, and no equivalent to Firefly for AI-generated content with commercial indemnification."
