## Step 1 — Page selection (15 min)

Run `main--site-scope--aemsites.aem.live` on the customer URL.

Pick exactly 3 pages:
- **Home** — always
- **Top-funnel / solutions / category page** — highest traffic non-home
- **Content-heavy page** — article, product detail, or resource page

Then run pre-flight on all 3 URLs before building anything:

```bash
bash yolo-preflight.sh [home-url] [top-funnel-url] [content-url]
```

All 3 must return HTTP 200 with no auth redirect. If any fail — swap for the next best alternative. Do not build against auth-walled pages.

---

## Step 2 — Baseline capture (30 min, automated)

Run all of these in parallel before touching the import pipeline:

```
Playwright Lighthouse run on all 3 URLs:
  → Performance score (mobile + desktop)
  → LCP, TBT, CLS values
  → Core Web Vitals pass/fail

Content-change timing simulation:
  → Screenshot current CMS authoring interface
  → Note the workflow: who approves, how many steps, time to live

Site scope artifact:
  → Template inventory (X templates found)
  → Block pattern frequency (top 5 patterns)
  → Complexity score (low / medium / high)
```

**Save all baseline numbers.** They are the "before" column in the results table.

---

## Step 3 — ExMod import pipeline (overnight or 2–3 hrs)

For React/SPA sites — Playwright render is always Step 0, no exceptions:

```bash
node scrape-rendered.js [url] → full-dom.html + screenshots
```

Scroll-simulate before capturing. Lazy-loaded sections won't appear otherwise.
Feed `full-dom.html` into the import pipeline — never raw HTTP.

Then run the full pipeline on all 3 pages:

```
/page-import chains automatically:
/scrape-webpage → /identify-page-structure → /page-decomposition
→ /authoring-analysis → /generate-import-html
```

**Block inventory check (parallel with scrape):**
- `/block-inventory` — existing project blocks
- `/block-collection-and-party` — Block Collection matches

Build only blocks with no existing match:

```
Wave 1 (parallel): /analyze-and-plan for each gap block
Wave 2 (parallel): /building-blocks → JS + CSS + UE model JSON per block
```

## Step 4 — DA + Universal Editor wiring (1 hr)

- `fstab.yaml` → DA content mount
- `component-definition.json`, `component-filters.json`, `component-models.json`
- Per-block UE model JSON for every block built in Step 3
- `da_write` all 3 pages → CDN preview triggered → published

**Scripted authoring flow for the demo:**
Change hero copy → update one card → publish → Lighthouse re-run. This is the live moment. Practice it.
